= Техническое решение для проекта Солидарный "умный огород"

== Введение

Проект "Солидарный 'умный огород'" направлен на создание системы, которая обеспечивает автоматизированный уход за растениями для занятых дачников, включая пенсионеров и работающих людей. Основная цель — минимизация усилий по уходу за растениями и сохранение урожая в условиях ограниченного времени.

== Техническое решение

Решение позволяет подключать устройства различного функционала, конфигурация осуществляется самим устройством, которое отправляет файл конфигурации ("манифест"). Основные компоненты включают:

- **Автополив с погодной компенсацией**: система автоматически регулирует полив в зависимости от погодных условий, что позволяет экономить ресурсы и предотвращает засыхание растений. Приведен лишь пример одного из возможных устройств. Уже разработаны и другие устройства, включая системы проветривания и мониторинга.
- **Контроль состояния**: датчики, которые отслеживают уровень воды, температуру и другие параметры, отправляют уведомления пользователям, позволяя им быть в курсе состояния своих растений.
- **Интуитивно понятный интерфейс**: фронтенд на Django обеспечивает простой и доступный интерфейс для управления системой, включая возможность удаленного контроля через мобильные устройства.

== Протокол файла конфигурации (манифеста)

Файл конфигурации (манифест) представляет собой JSON-документ, который содержит настройки и параметры устройства. Структура файла включает:

- **device_id**: уникальный идентификатор устройства.
- **device_type**: тип устройства (например, датчик, исполнительный механизм).
- **configuration**: объект с параметрами конфигурации, специфичными для данного устройства.
- **firmware_version**: версия прошивки устройства.
- **last_update**: временная метка последнего обновления конфигурации.

Файл отправляется устройством на платформу ThingsBoard при инициализации и может обновляться по мере необходимости. Платформа обрабатывает полученные данные и сохраняет их для дальнейшего использования.

== Общая архитектура

Сигнал от устройства проходит через Wi-Fi или GSM-сеть к платформе ThingsBoard, где обрабатывается и сохраняется. Устройство отправляет данные о состоянии и получает команды управления через API ThingsBoard. Платформа, в свою очередь, отправляет команды обратно на устройство, обеспечивая двустороннюю связь и управление. Логика взаимодействия включает в себя:

- **Отправка данных**: Устройство отправляет данные о состоянии (например, уровень воды, температура) в формате JSON на платформу ThingsBoard.
- **Получение команд**: Платформа отправляет команды управления (например, включение/выключение полива) обратно на устройство, что позволяет пользователю управлять системой удаленно.
- **Мониторинг и уведомления**: ThingsBoard обрабатывает данные и может отправлять уведомления пользователю о состоянии системы, что позволяет оперативно реагировать на изменения.

== Пример устройства: Контроллер на базе ESP32

Контроллер ESP32 является примером уже разработанного устройства, которое реализует описанную логику. Он использует модульную архитектуру, включая инициализацию периферии, основной цикл работы и обработку RPC-команд. Контроллер может управлять актуаторами, собирать данные с датчиков и взаимодействовать с платформой ThingsBoard для передачи телеметрии и получения команд управления.

== Заключение

Данное техническое решение не только решает актуальные проблемы дачников, но и создает основу для дальнейшего развития и масштабирования проекта. Мы уверены, что с получением гранта сможем реализовать этот проект и сделать его доступным для широкой аудитории.
